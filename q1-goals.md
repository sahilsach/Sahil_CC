# Q1 2026 Goals - Product Team

## Theme: "TA Excellence"

This quarter we're doubling down on teaching assistant workflows. Our research shows TAs are the bottleneck for grading speed AND consistency. If we nail TA tools, we unlock value for everyone.

---

## Goal 1: Improve Grading Consistency
**Owner:** Product Team
**Key Result:** Reduce grading variance between TAs by 40%

Current state: When multiple TAs grade the same rubric item, scores vary by an average of 15%. This creates student complaints and extra work for professors who have to normalize grades.

Initiatives:
- [ ] Launch calibration exercise improvements
- [ ] Add real-time consistency alerts
- [ ] Build TA grading comparison dashboard

---

## Goal 2: Increase TA Adoption
**Owner:** Product + Customer Success
**Key Result:** 80% of TAs actively using GradeFlow within 2 weeks of course start

Current state: Only 60% of TAs assigned to GradeFlow courses actively use the platform. Many still grade on paper or use workarounds.

Initiatives:
- [ ] Redesign TA onboarding flow
- [ ] Create TA-specific training materials
- [ ] Add "TA view" that hides professor-only features

---

## Goal 3: Surface Workload Imbalances
**Owner:** Product Team
**Key Result:** Professors can identify overloaded TAs before burnout

Current state: Professors have no visibility into how grading work is distributed. Some TAs are overwhelmed while others have capacity.

Initiatives:
- [ ] Build workload visibility dashboard
- [ ] Add grading time estimates
- [ ] Create workload alerts for professors

---

## Goal 4: Ship AI Feedback Assistant (GA)
**Owner:** Product + Engineering
**Key Result:** 50% of active courses using AI feedback suggestions

Current state: Beta launched Q3 2024. Feedback is positive but adoption is limited. Need to move from beta to general availability.

Initiatives:
- [ ] Address top 10 beta feedback items
- [ ] Build professor controls for AI tone/style
- [ ] Create "suggested comments" library

---

## Dependencies

- **Engineering capacity**: Goals 1 and 3 require significant frontend work
- **Data science**: Consistency metrics need new analysis pipeline
- **Customer success**: Goal 2 requires updated training materials

## Risks

- Spring semester starts mid-January - limited time for user research
- AI feedback work may expand scope based on beta learnings
- New PM starting (that's you!) - onboarding time
